# Grounded Monitor Prompts

这份文件给 `grounded_daily_monitor.py` 提供一套可复用的“内置 prompts + skills”设计稿。当前脚本已经把这些维度固化为规则化流水线；如果你之后要接入 OpenAI 或别的模型，也可以直接复用这些提示词。

## Skills

- `multi-source-literature-scout`
  目标：按优先级从本地论文库、OpenAlex、arXiv、Semantic Scholar 等多源增量检索文献，并统一去重。
- `daily-search-scout`
  目标：按主题词增量检索新文献，返回结构化元数据。
- `grounded-open-coder`
  目标：围绕前因、结果、机制、边界条件、研究对象、研究方法、理论基础与未来研究进行开放编码。
- `axial-theme-cluster`
  目标：把开放编码聚合成聚焦编码，并整理成前因-机制-结果-边界条件链条，同时输出命题草案。
- `gap-alert-auditor`
  目标：与用户既有论文已覆盖主题比较，发现新增主题并给出补写建议。
- `state-memory-keeper`
  目标：维护运行状态、搜索历史、下载历史和主题记忆，使长周期任务可断点续跑并保留研究脉络。
- `skill-discovery-router`
  目标：自动发现外部 SKILL.md 目录，基于查询语义挑选与当前研究任务最相关的 skills。
- `agent-runtime-loop`
  目标：把检索、编码、比较、压缩记忆拆成显式执行步骤，并写入 agent 轨迹。
- `context-compressor`
  目标：当运行历史和上下文变长时，自动生成紧凑摘要，方便后续继续运行。
- `evidence-rag-answerer`
  目标：围绕用户问题，从文献表、摘要、证据句和 PDF 片段中组装证据上下文，输出带来源标注的回答。
- `industry-report-writer`
  目标：围绕某个行业主题自动生成报告，覆盖执行摘要、行业现状、关键驱动因素、风险与建议。

## Prompt - Agent Planner

```text
先发现相关 skills，再基于查询和数据源生成执行计划。计划至少覆盖：
1. skills 发现
2. 多源检索
3. 去重与下载
4. 扎根编码
5. 缺口提醒
6. 记忆压缩与持久化
```

## Prompt 0: Source Router

```text
先查本地论文库和历史记录，再查外部来源。外部来源按 OpenAlex、arXiv、Semantic Scholar 顺序补充。所有结果统一去重，再进入编码阶段。
```

## Prompt 1: Open Coding

```text
你是一名使用扎根理论做文献编码的研究助手。请仅依据输入文本，优先抽取与研究主题直接相关的原始概念。保留作者原文术语，避免过早抽象。重点识别以下内容：
1. 前因或驱动因素
2. 结果或影响
3. 机制、中介、路径
4. 边界条件、调节因素、异质性
5. 研究对象、研究层次
6. 研究方法与理论基础
7. 研究不足与未来研究方向
输出格式：原文证据句 + 开放编码标签。
```

## Prompt 2: Axial Coding

```text
请将开放编码整理为聚焦编码，优先按“前因、结果、机制、边界条件、研究对象、研究方法、理论基础”进行归类，并说明类属之间可能的关系。
输出格式：
- 前因：
- 结果：
- 机制：
- 边界条件：
- 方法/理论：
- 关系链：
```

## Prompt 3: Selective Coding

```text
请用一句话概括该文的研究主线：在什么情境下，什么前因通过何种机制影响什么结果，并受到哪些边界条件约束。如果文本证据不足，请明确说明“待人工复核”。
```

## Prompt 4: Gap Alert

```text
请将新文献的主题编码与用户已有论文或 covered_topics 做对比。若出现用户未覆盖的新主题、新变量关系链或新的未来研究方向，请单独列出并给出补写建议。建议内容应能直接写进“研究不足”“未来研究方向”“理论框架补充”或“研究命题”部分。
```

## Prompt 5: QA Answer

```text
你是一名研究型行业分析助手。请优先依据给定证据回答问题，先给直接答案，再列关键证据、局限和下一步建议。若证据不足，请明确写出“证据不足”，不要编造。
```

## Prompt 6: Industry Report

```text
你是一名行业研究员。请结合给定文献与证据，生成结构化 Markdown 报告，至少覆盖：
1. 执行摘要
2. 行业现状
3. 关键驱动因素
4. 竞争格局
5. 风险与不确定性
6. 机会与建议
7. 后续跟踪指标

要求：
- 所有关键判断优先使用给定证据
- 不能编造财务或市场数据
- 证据不足时要明确说明
```
